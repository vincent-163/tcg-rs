## Context

当前 tcg-rs 的 TB 链接只有"软链接"——执行循环通过 `jmp_target[slot]`

## 设计

### 核心思路

参考 QEMU `cpu-exec.c` 和 `tb-maint.c`，tcg-rs 是单线程的，
因此不需要自旋锁和原子 CAS，但需要完整的链接/解链/失效协议。

### 新增 TB 字段（`core/src/tb.rs`）

```rust
/// 出边目标 TB 索引（slot 0/1），None = 未链接。
pub jmp_dest: [Option<usize>; 2],

/// 入边链表头：(source_tb_idx, slot) 对。
/// 记录哪些 TB 的 goto_tb 跳转到了本 TB。
pub jmp_list: Vec<(usize, usize)>,
```

QEMU 用 LSB 编码的侵入式链表，tcg-rs 单线程可以直接用 `Vec`。

### W^X 处理

QEMU x86-64 不需要 icache flush，但 tcg-rs 使用 `mprotect` 做 W^X。
每次 patch 都切换 RW→RX 太慢。改为：

**方案：使用 `mmap` 双映射（split W/X）**——太复杂。

**方案：放宽为 RWX**——不安全但简单。

**方案：批量 patch**——在执行循环的 `tb_find` 返回后、执行前做 patch，
然后切换回 RX。但这意味着每次循环都要切换两次 mprotect。

**最佳方案：保持 RWX 在运行时**。代码缓冲区在 `tb_gen_code` 结束后
设为 RX，但 patch 时需要临时 RW。由于 x86-64 不区分 icache/dcache，
可以直接用 `mprotect(RWX)` 一次性设置，或者在 patch 时临时切换。

实际上，最简单的方案是：**代码缓冲区始终保持 RWX**。在 `CodeBuffer::new`
中直接分配 `PROT_READ | PROT_WRITE | PROT_EXEC`，去掉 `set_writable`/
`set_executable` 的调用。这在 x86-64 Linux 上是安全的（没有 W^X 强制），
且是 QEMU 在非 split-wx 模式下的默认行为。

### tb_add_jump（`exec/src/exec_loop.rs`）

在执行循环中，当 `exit_val` 为 0 或 1（chain slot）时：

```rust
fn tb_add_jump(env: &mut ExecEnv<B>, src: usize, slot: usize, dst: usize) {
    let src_tb = env.tb_store.get(src);
    // 只有 goto_tb 槽位存在时才链接
    let jmp_off = match src_tb.jmp_insn_offset[slot] {
        Some(off) => off as usize,
        None => return,
    };
    let dst_tb = env.tb_store.get(dst);
    if dst_tb.invalid { return; }

    // 已经链接到同一目标？跳过
    if env.tb_store.get(src).jmp_dest[slot] == Some(dst) { return; }

    // 修补跳转指令
    let src_host = src_tb.host_offset;
    let dst_host = dst_tb.host_offset;
    let abs_jmp = src_host + jmp_off;
    let abs_dst = dst_host;
    env.backend.patch_jump(&mut env.code_buf, abs_jmp, abs_dst);

    // 记录出边
    env.tb_store.get_mut(src).jmp_dest[slot] = Some(dst);

    // 记录入边（反向链接）
    env.tb_store.get_mut(dst).jmp_list.push((src, slot));
}
```

### tb_reset_jump（`exec/src/tb_store.rs`）

```rust
fn tb_reset_jump(tb_store: &TbStore, code_buf: &mut CodeBuffer,
                 backend: &B, src: usize, slot: usize) {
    let tb = tb_store.get(src);
    if let (Some(jmp_off), Some(reset_off)) =
        (tb.jmp_insn_offset[slot], tb.jmp_reset_offset[slot])
    {
        let abs_jmp = tb.host_offset + jmp_off as usize;
        let abs_reset = tb.host_offset + reset_off as usize;
        backend.patch_jump(code_buf, abs_jmp, abs_reset);
    }
}
```

    self.tbs[tb_idx].invalid = true;

    // 1. 从哈希表移除（已有）
    // ...

    // 2. 解除所有入边：遍历 jmp_list，重置每个源 TB 的跳转
    let jmp_list = std::mem::take(&mut self.tbs[tb_idx].jmp_list);
    for (src, slot) in jmp_list {
        tb_reset_jump(self, code_buf, backend, src, slot);
        self.tbs[src].jmp_dest[slot] = None;
    }

    // 3. 解除自己的出边
    for slot in 0..2 {
        if let Some(dst) = self.tbs[tb_idx].jmp_dest[slot].take() {
            self.tbs[dst].jmp_list.retain(|&(s, n)| !(s == tb_idx && n == slot));
        }
    }
}

### 执行循环修改（`exec/src/exec_loop.rs`）

```rust
match exit_val {
    v @ 0..=1 => {

```

注意：启用硬链接后，`jmp_target` 软缓存不再需要——goto_tb 直接跳转
到目标 TB，不会返回执行循环。只有首次执行（未链接时）才会返回。
因此可以移除 `jmp_target` 字段，简化代码。

`exit_target` 仍然需要保留，用于间接跳转（JALR）的软缓存。
